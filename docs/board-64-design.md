# 当前 64 格版棋盘总设计

> 本文基于当前代码实现整理，不再采用旧的 60 格方案。

## 1. 棋盘总览

- **总格数**：64
- **布局方式**：17×17 网格外圈一圈，共 64 格
- **行进方向**：顺时针
- **四个角**：
  - `#0` 起点
  - `#16` 监狱/探访
  - `#32` 免费停车
  - `#48` 入狱

当前棋盘由服务端 `mapData` 作为唯一格子定义来源，客户端通过同一套索引和坐标规则渲染。

## 2. 当前代码中的棋盘真源

- 服务端棋盘定义：`server/game/data/mapData.ts`
- 地产组与租金表：`server/game/data/propertyData.ts`
- 棋盘封装与查询：`server/game/Board.ts`
- 客户端格子世界坐标：`src/utils/cellPositions.ts`
- 3D 棋盘渲染入口：`src/components/Board3D/GameCanvas.tsx`
- 单格渲染：`src/components/Board3D/Cell.tsx`
- 特殊格标识：`src/components/Board3D/SpecialMarker.tsx`
- 建筑渲染：`src/components/Board3D/Building.tsx`

## 3. 64 格索引分布

### 底边：`#0 ~ #15`
- 左下角起点开始，沿底边向右

### 右边：`#16 ~ #31`
- 右下角起，沿右边向上

### 顶边：`#32 ~ #47`
- 右上角起，沿顶边向左

### 左边：`#48 ~ #63`
- 左上角起，沿左边向下回到起点上方

## 6. 客户端 3D 渲染链路

1. `server/game/data/mapData.ts` 定义 64 格格子数据。
2. `src/utils/cellPositions.ts` 把 64 格映射到 17×17 外圈的世界坐标。
3. `src/components/Board3D/GameCanvas.tsx` 统一遍历 `mapData` 渲染。
4. `Cell.tsx` 负责单格外观、悬浮和点击。
5. `SpecialMarker.tsx` 给功能格、税格、车站、公用事业加立体标识。
6. `Building.tsx` 在地产格上方渲染房屋 / 酒店。

## 7. 规则与实现上的一致性说明

- 服务端 `Board.count` 直接来自 `mapData.length`，因此棋盘规模跟随数据源。
- 客户端与服务端都已经切到 64 格，后续维护以 64 格为唯一标准。
- 旧的 60 格设计已废弃，不应再作为新改动依据。

## 8. 仍保留的 60 含义

以下数字中的 `60` 不是棋盘格数，不应改动：
- 回合上限 `MAX_ROUNDS = 60`
- 金额数值、租金、卡牌效果
- 事件列表窗口等 UI 截断逻辑
